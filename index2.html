🧩 What is a Fragment?

🧩

Modular Component

A fragment is a reusable piece of your app's UI that can be combined with other fragments to build a complete screen.

🏠

Mini-Activity

Has its own layout, lifecycle, and input events, but must be hosted by an activity.

📱 Activity as Container

🧩 Fragment A (Header)
🧩 Fragment B (Content)
🧩 Fragment C (Footer)
👨‍🏫 Teacher's Note: Explain fragments as LEGO blocks - each piece has its own shape and function, but they all connect to the same baseplate (Activity) to create something bigger!

❓ Why Do We Need Fragments?

🔄

Dynamic UI

Change screen content at runtime without starting new activities

📱💻

Screen Adaptation

Create different layouts for phones and tablets using same code

🔄

Reusability

Use same fragment in multiple activities or multiple times

Feature Without Fragments With Fragments
Tablet Support ❌ Difficult ✅ Easy

Dynamic UI ❌ Static ✅ Flexible

📝 Fragment Transactions

🔧 Transaction Methods

  • ✅ add() - Add fragment to container
  • ✅ remove() - Remove fragment
  • ✅ replace() - Replace existing fragment
  • ✅ hide()/show() - Toggle visibility
  • ✅ attach()/detach() - Connect/disconnect
// Complete Transaction Example
FragmentManager manager = 
    getSupportFragmentManager();
FragmentTransaction transaction = 
    manager.beginTransaction();

// Add animation
transaction.setCustomAnimations(
    R.anim.slide_in_right,
    R.anim.slide_out_left
);

// Replace fragment
transaction.replace(R.id.container, 
    new MyFragment());

// Add to backstack
transaction.addToBackStack(null);

// Commit
transaction.commit();
👨‍🏫 Teacher's Note: Always use addToBackStack() if you want users to navigate back. Without it, pressing back will close the app!

📞 Fragment Communication

🔷 Method 1: Interface

Best practice for Activity → Fragment communication

// In Fragment
interface OnItemSelected {
    void onItemSelected(String data);
}

// In Activity
public class MainActivity implements 
    OnItemSelected {
    @Override
    public void onItemSelected(String data) {
        // Handle data
    }
}

🔶 Method 2: ViewModel

Best for Fragment → Fragment communication

// Shared ViewModel
public class SharedViewModel extends ViewModel {
    private MutableLiveData<String> selected = 
        new MutableLiveData<>();
    
    public void select(String item) {
        selected.setValue(item);
    }
}

💡 Best Practices

🗃️

Use Back Stack

Always add transactions to back stack for natural navigation

🎨

Custom Animations

Use setCustomAnimations() for smooth transitions

⚡

Lazy Loading

Load data in onViewCreated(), not onCreateView()

🧹

Clean Up

Clear references in onDestroyView()

🔄

Reuse Fragments

Create factory methods for fragment instances

📱

Responsive Design

Use different layouts for phones/tablets

⚠️ Common Pitfalls to Avoid:
  • ❌ Don't store Activity context in static variables (memory leaks)
  • ❌ Don't perform transactions after activity state is saved
  • ❌ Don't assume fragment is still attached to activity
  • ❌ Don't forget to check savedInstanceState in onCreate()
